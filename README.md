首先下载安装好git后，右键你的unity项目地址，选择git bush here
在出来的窗口中依次输入,输完一行就按回车
git init
git lfs install
git remote add origin https://github.com/zhang5120240703/TiaoZhanBei_Competition
然后重启Visual Studio，选择上方导航栏中的git-提交或储存-查看全部提交-传入那一行的拉取


在提交本地更改时，选择要提交的文件，然后点查看全部提交，选择本地分支，提取拉取同步推送的意思分别是：
1. 推送（Push）
含义：将本地已提交的代码，上传到远程仓库（如 GitHub）。
场景：本地完成开发并提交后，把代码同步到远程，让其他人可以获取。
2. 提取（Fetch）
含义：从远程仓库下载最新代码的元数据（仅更新本地的远程分支记录，不修改本地工作代码）。
场景：想查看远程有哪些新提交，但暂时不想合并到本地代码。
3. 拉取（Pull）
含义：等价于 “Fetch + 合并”—— 先下载远程最新代码，再自动合并到本地当前分支。
场景：需要将远程的最新代码同步到本地工作目录，直接用于开发。
4. 同步（Sync）
含义：VS 的快捷操作，通常等价于 “拉取（Pull） + 推送（Push）”（先拉取远程更新，再推送本地提交）。
场景：快速完成本地与远程的双向同步（需确保本地无冲突）。


然后在提交时记得完整详细的写好此次改动的内容，记得加上前缀，比如fix:修复xxxxbug，常用的前缀有：
1. feat:	新增功能（feature）	feat: 新增用户登录模块
2. fix:	修复 bug	fix: 解决订单支付失败问题
3. docs:	文档修改（仅文档）	docs: 更新API接口说明
4. style:	代码格式调整（不影响逻辑）	style: 统一代码缩进
5. refactor:	代码重构（不增功能 / 修 bug）	refactor: 优化订单处理逻辑
6. test:	新增 / 修改测试代码	test: 补充用户模块单元测试
7. chore:	构建 / 工具 / 依赖等配置修改	chore: 升级npm依赖包
8. perf:	性能优化	perf: 优化列表渲染速度
9. revert:	回滚之前的提交	revert: 撤销feat: 新增登录模块
